# Contributing

Contributions that preserve the classroom and portability goals are welcome.

## Before opening a pull request

1. Keep each applet usable as one HTML file.
2. Preserve English and Chinese interface coverage.
3. Add or update algorithmic tests when behavior changes.
4. Run `python tools/release_check.py`.
5. Run `python tools/browser_qa.py --no-screenshots`.
6. Explain any intentional simplification in the teacher notes.

Use normal software-development language in public files. Internal research notes and model-specific workflow records do not belong in the public release tree.


## Learner-facing writing

Public applet copy follows [`CONTENT_STYLE_GUIDE.md`](CONTENT_STYLE_GUIDE.md), including canonical naming, abbreviation rules, scenario structure, bilingual parity, and toolbar hierarchy.
