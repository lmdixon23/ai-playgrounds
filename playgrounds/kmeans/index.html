↓ Read the explanation

Suppose you plotted the screen width and height of every device that visits a website. You'd probably see three clumps: smartphones (small), tablets (medium), and laptops (large). Getting an algorithm to find those clumps without being told where they are is called clustering. K-means is the simplest and most-used clustering algorithm.

How K-Means Works

K-means starts with a question you have to answer for it: how many clusters? Call that number k. Then place k centroids somewhere in the plane — randomly, by the "Farthest Point" rule, or by clicking the canvas yourself.

The algorithm then alternates two steps until nothing changes:

  • Reassign Points. For every data point, find which centroid is closest, and assign the point to that centroid's cluster.
  • Update Centroids. For each centroid, move it to the mean (average position) of all the points currently assigned to it.
The shaded regions on the canvas are called a Voronoi diagram: each region is the set of all locations that are closer to that centroid than to any other. In the Reassign Points step, every data point gets the color of whatever Voronoi region it's sitting in.

After Update Centroids, the Voronoi cells redraw because the centroids moved. Some points are now in a different-colored region than their own color — those are the points that will switch clusters at the next Reassign step. When no point switches, the algorithm has converged.

Why does it always converge?

The total sum of squared distances from each point to its assigned centroid is what k-means is minimizing. Both steps reduce that sum: the Reassign step lowers it by sending points to their closest centroid, and the Update step lowers it by moving the centroid to the optimum position for its current points. The sum can't go on decreasing forever because there are only finitely many ways to assign n points to k clusters. So k-means is guaranteed to halt.

When K-Means Goes Wrong

Convergence is guaranteed, but a good answer is not. Try the Gaussian Mixture dataset with k = 4 instead of the correct k = 3, and run it a few times. You'll usually see two centroids splitting a single blob between them while a third centroid covers two blobs. Which blob gets split depends on where the centroids start — that's why people typically run k-means many times with different random starts and keep the lowest-cost result.

Try Smiley Face or Pimpled Smiley with k = 4. K-means cannot find the eyes, nose, and mouth as separate clusters because the mouth is a long curved arc, not a round blob. K-means' core assumption is that clusters are roughly spherical and roughly the same size — and the Smiley deliberately violates both.

K-means doesn't see your data the way you do. It sees a sum of squared distances, and any cluster shape that doesn't look like a round blob will get butchered.

How to Initialize the Centroids

Initialization matters more than people expect. The three strategies on the canvas demonstrate a real progression:

  • Random: pick k data points uniformly at random and use those as initial centroids. Simple and bad — if k = 5 and your data has 5 true clusters, the probability that your 5 random picks all land in different clusters is roughly k!/kk, which is about 4% for k = 5 and 0.04% for k = 10.
  • Farthest Point: pick the first centroid at random, then pick each next centroid to be the data point whose distance to the nearest existing centroid is largest. This spreads centroids across the dataset. Try it on the Packed Circles dataset with k = 6 and watch how often it lands one centroid in each circle.
  • I'll Choose: click anywhere on the canvas to place the centroids yourself. This is the most pedagogical option — put one centroid in a stupid place and watch it die.

The most common modern strategy, k-means++, isn't in this visualization but is worth knowing about: it works like Farthest Point but picks the next centroid randomly, with probability proportional to the squared distance from the nearest existing centroid. That gives you the spread-out behavior of Farthest Point without the tendency to land on outliers (which are, by definition, far from everything).

How Many Clusters?

k is a parameter you have to supply, and there's no automatic way to know the right answer. When you can visualize the data — like here — you can often see how many clusters there should be. In real applications the data lives in hundreds of dimensions and you can't see it, so people use heuristics like the "elbow method" (plot total cost vs k, look for the kink) or domain knowledge ("we want exactly 5 customer segments because that's what the marketing team can act on").

What This Simplifies

This applet uses Euclidean distance, equal-weight points, no noise points, and the standard Lloyd's algorithm. Real implementations vary on every dimension: k-medoids replaces the mean with an actual data point (more robust to outliers); fuzzy c-means assigns each point partially to multiple clusters; Gaussian mixture models treat each cluster as a probability distribution rather than a hard region; DBSCAN abandons the centroid idea entirely and grows clusters from dense regions of points (which handles the Smiley face that k-means fails on).

For teachers

Curriculum: Unsupervised Learning & Representation. Learning objectives: clustering as structure-finding; the two-step k-means iteration; sensitivity to initialization; failure modes for non-spherical or unequal clusters.

Pre-exploration prompts:

  • "Unsupervised" means no labels. How might a computer group things without being told the groups?
  • If you wanted to split a crowd into "k" friend groups, what would you measure?
  • What does it mean for an algorithm to "converge"?

Post-exploration prompts:

  • Pick "I'll Choose" and put all k centroids in one corner. Run k-means. Does it ever recover the true clusters? Why or why not?
  • Run k-means with k = 2 on Smiley Face. Now k = 4. Now k = 6. Which value gives the most "Smiley-like" answer? Does ANY value find the actual face features?
  • On Packed Circles with k = 6, compare Random vs Farthest Point initialization. Which one more reliably finds the six clusters?
  • Add a few outlier points far from the main data. What happens to the centroids? What does this say about k-means and outliers?

Common misconceptions defeated: "k-means finds the 'right' number of clusters" (no — you supply k); "k-means works for any cluster shape" (no — it assumes round-ish, equal-sized clusters); "lower cost = better model" (only comparing same k; smaller k always has higher cost).

What's under the hood here vs. industry tools: This applet implements Lloyd's algorithm, the canonical assign-then-update loop students should know. Production libraries like scikit-learn's KMeans default to Elkan's variant (skips redundant distance calculations using the triangle inequality) and k-means++ initialization (probabilistically biased toward spreading initial centroids apart, like our "Farthest Point" but with randomness). Same algorithm family, same convergence guarantees — those defaults are speed/quality refinements, not a different method.
