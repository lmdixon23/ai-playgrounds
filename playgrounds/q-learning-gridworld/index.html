🧮 Linear function approximation (bridge to deep RL)

Instead of a Q-table with one entry per (state, action), we approximate Q(s,a) = w_a · φ(s). For a 7×9 grid that's 252 numbers tabular vs only 36 with this 9-tile coding (4 actions × 9 tile weights). The trade-off: weights generalize across cells — fewer parameters, but the model can't represent arbitrary value functions.

Update rule: w_a ← w_a + α · (target − Q(s,a)) · φ(s). One TD step changes ALL the active tile weights at once.

↓ Read the explanation

Reinforcement learning answers a question that supervised learning can't: how does an agent learn to act when there's no teacher to tell it the right answer? Q-learning's answer is simple and powerful — try things, see what reward you get, and remember.

The Q-table

The agent maintains a number Q(state, action) for every cell of the grid and every action it might take from that cell. This number estimates "if I take this action from this state and then play optimally forever after, how much total reward will I collect?" At the start, every Q-value is zero — the agent knows nothing.

The Bellman update

Each time the agent moves, it observes the reward and the cell it landed in. It then nudges the Q-value of the action it just took toward what it just learned:

Q(s, a) ← Q(s, a) + α [ r + γ · maxa' Q(s', a') − Q(s, a) ]

The bracketed quantity is the temporal-difference error — the difference between what the agent expected (Q(s, a)) and what it actually saw (the reward plus the discounted best value of the next state). α controls how much the agent moves toward the new estimate; γ controls how much it cares about that "best value of the next state" versus just the immediate reward.

The Q-values literally spread outward from the goal. Watch the green color expand cell by cell as the agent stumbles upon the goal a few times — each visit nudges the Q-value of the cell just outside, which on the next visit nudges the cell outside that, and so on. That's "reward propagating backward through experience."

The algorithm family

Q-learning is one member of a family of tabular value-based methods. The algorithm dropdown above lets you compare four of them on the same gridworld. They share the same Q-table and the same overall loop; they differ in what goes into the bracketed TD target:

  • Q-learning uses r + γ · maxa' Q(s', a') — always the best possible next action. Off-policy: the policy used to act (ε-greedy) is different from the policy used to evaluate (greedy). Bold, optimistic, sometimes overestimates.
  • SARSA uses r + γ · Q(s', a'), where a' is the action the agent actually picks next (using ε-greedy). On-policy: the policy used to act is the same one being evaluated. Tends to be more cautious near cliffs because it accounts for the chance of taking a risky exploration step.
  • Expected SARSA uses r + γ · Σa' π(a'|s') · Q(s', a') — the expected next-state value under the current policy. A smoother on-policy alternative to SARSA; lower variance because it averages over all possible next actions instead of just one sampled one.
  • Monte Carlo (first-visit control) waits until the end of an episode, then updates each visited (s, a) toward the full return Gt from that point forward. No bootstrapping at all — pure samples. The "Step" button is disabled in MC mode because MC updates only happen at episode end.

The fifth option, Q-learning (Softmax), switches the exploration strategy from ε-greedy to a temperature-weighted action distribution: each action is picked with probability proportional to exp(Q(s, a) / τ). Lower τ = greedier; higher τ = more random. Unlike ε-greedy (which explores uniformly when it explores), Softmax preferentially explores actions that look promising, which can speed up learning in some environments and slow it down in others.

On the default gridworld with a pit, SARSA learns a path that stays a few cells away from the pit; Q-learning learns the shortest path even if it walks right next to the pit. Why? SARSA accounts for the chance the agent will randomly fall in during exploration; Q-learning assumes perfect greedy execution and doesn't. This is the canonical "Cliff-walking" insight from Sutton & Barto Ch 6.

Exploration vs. exploitation

If the agent always took the best-known action, it would never discover that other actions might be better. That's exploitation only. If it always took random actions, it would never get good at anything. That's exploration only. Q-learning's standard fix is the ε-greedy policy: with probability ε take a random action, otherwise take the best one known so far. Watch what happens when you slide ε all the way to 0 — the agent gets stuck in whatever wrong path it happened to find first.

The role of the step penalty

Without a small per-step penalty, the agent has no reason to find a short path — just any path that eventually reaches the goal. Slide step reward to 0 and watch the agent learn meandering, beautiful, useless routes. The −0.04 default is borrowed from Russell & Norvig's textbook gridworld and is a real engineering knob in production RL.

What this simplifies

Real RL uses neural networks (deep Q-learning, DQN) when the state space is too big to fit in a table — chess has more states than atoms in the observable universe. Real RL also has to handle partial observability, continuous actions, and credit assignment over hundreds of steps. But the heart is exactly what you see here: a Bellman update, propagating signal backward through experience.

For teachers

Curriculum: Reinforcement Learning & Decision-Making. Learning objectives: state-action values; the Bellman update; on-policy vs ε-greedy exploration; how delayed rewards propagate backward through a value function.

Pre-exploration prompts:

  • How would you teach a dog a trick using only treats — no words?
  • If the goal is far away and every step costs a tiny bit, how do you decide which way to go?
  • What's the difference between "always trying the best-known action" and "sometimes trying something new"?

Post-exploration prompts:

  • Run "Train continuously" with defaults. Watch the green spread out from the goal. What does that show?
  • Set ε = 0 and reset. Train. Does the agent ever find the goal? Why or why not?
  • Set γ = 0 and train. What happens to the heatmap far from the goal? Why?
  • Set step reward = 0 and train. What changes about the paths the agent learns?
  • Add a pit between the agent and the goal. Watch the Q-values around it. How does the agent learn to avoid it?

Common misconceptions defeated: "RL agents have a plan" (no — they have a value table that implicitly defines a plan); "more exploration is always better" (no — too much ε prevents the policy from improving); "the agent sees the whole grid" (in this demo it does, but in real RL the agent only sees its own state).
