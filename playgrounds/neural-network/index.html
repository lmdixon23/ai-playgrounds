↓ Read the explanation

A neural network is just a stack of "carve a line, bend it, carve another line, bend it" — repeated layer after layer until the function is flexible enough to do what you want. This applet trains one on a 2D classification problem so you can watch the decision boundary bend in real time as the model learns.

What "layer" means here

Each layer takes the previous layer's outputs and computes activate(W·x + b): a linear combination of its inputs (the matrix W and bias b), then a non-linear "activation function" applied element-wise. The first hidden layer turns the 2D input into some new feature space; the second hidden layer can carve that feature space further; the final layer collapses everything to one number — the predicted probability of class B.

Why non-linearity is non-negotiable

Pick the "linear" activation and try to fit two moons. The model stalls as a single straight line no matter how many hidden layers you add. The reason is algebraic: a composition of linear functions is still a linear function. To bend the decision boundary you need a non-linear step somewhere, and that's exactly what ReLU, tanh, and sigmoid provide.

Take "linear" activation with any architecture and try XOR. The model fails forever. Take ReLU with hidden sizes 0/0 and try moons — it also fails. Take ReLU with hidden sizes 6/4 and try moons — it works. The non-linearity matters as much as the depth, and depth matters only because non-linearity exists.

What you're watching when you press train

The applet does full-batch stochastic gradient descent: for each of the 200 training points, it computes the prediction, the cross-entropy loss, then propagates gradients back through every layer to update every weight. Each "epoch" is one pass over all 200 points. The loss curve on the right is the sum of all per-point losses, divided by 200.

Common failure modes you can produce here

  • Too high learning rate — slide LR to 0.3. Loss oscillates or diverges.
  • Too low learning rate — slide LR to 0.001. Loss falls so slowly you'll lose patience.
  • Not enough capacity — hidden=0, dataset=moons. Loss plateaus.
  • Wrong activation for the data — sigmoid on circles often gets stuck in a bad local minimum because gradients vanish far from zero.

What this simplifies

Real neural networks use mini-batch SGD (batches of 32 or 256, not the whole dataset), sophisticated optimizers (Adam, AdamW) with momentum and adaptive per-parameter learning rates, hundreds or thousands of layers, regularization (dropout, weight decay, batch norm), and millions of parameters. But the core loop — forward pass, compute loss, backward pass, update weights — is exactly what runs here.

For teachers

Curriculum: Neural Network Fundamentals. Layers and activations; gradient descent; capacity vs data size; the XOR-as-non-linearity-litmus-test. Input-feature toggles (x², x·y, sin) let students see that feature engineering changes what the network can separate; the network-graph view renders the weights as connections.

Pre-exploration prompts:

  • Can you draw a single straight line that separates the XOR points? Why or why not?
  • What might a "hidden layer" do that the inputs and outputs don't?
  • If a model has more parameters, is it always better?

Post-exploration prompts:

  • Activation = "linear", any architecture, dataset = moons. Train. Why does this stall as a straight line?
  • Hidden sizes 0/0, ReLU, XOR. Train. Why is the boundary still a line?
  • Hidden 8/8, ReLU, circles. Train. Watch the boundary unfold from straight → curved → closed. What's happening?
  • Crank LR to 0.3. What goes wrong, and why?

Common misconceptions defeated: "deeper is always better" (training instability says no); "the model sees the dataset name" (no — only the (x, y, label) tuples); "a hidden layer alone is enough" (no — without non-linearity it collapses).
